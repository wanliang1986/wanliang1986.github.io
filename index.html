immutable.js

发表于 2020-08-26 | 分类于 前端 |

介绍

immutable与react同期出现，但没有被默认放到react工具集中。它内部实现了一套完整的数据持久化体系，同时还有很多比较容易使用的数据类型，
比如 List,Map,Set等
其中，最重要的三种数据结构：

  • Map: 键值对集合，对应于Object

  • List: 有序可重复的列表，对应于Array

  • Set：无序不可重复的列表
    immutable Data 一旦创建，就不能再被更改的数据，对immutable对象的任何修改和添加删除操作，都会返回一个新的immutable对象，这是其最本质的特点

    为什么数据不能被改变，这是由于我们所运用的场景，比如redux来决定的， redux的基本原则是redux里的数据都是只读的，所以如果在redux中要对数据进行操作，那么就需要对原数据进行拷贝操作，这时候就可以用到immutable；另一种情况是在一个复杂的业务场景中，存在一个共享数据，当多个业务场景都可以操作这个数据时，就会出现不可控的结果。

深拷贝与浅拷贝

发表于 2020-06-30 | 分类于 前端 |

在实际开发项目中，我们经常会对从后台获取到的数据进行处理，当我们不想改变原始数据的情况时，我们就需要对原始数据进行拷贝。这里对深拷贝与浅拷贝来进行解释及说明

深拷贝与浅拷贝

深拷贝和浅拷贝最根本的区别在于是否真正获取一个对象的复制实体，而不是引用。
假设B复制了A，修改A的时候，看B是否发生变化：
如果B跟着也变了，说明是浅拷贝（修改堆内存中的同一个值）
如果B没有改变，说明是深拷贝（修改堆内存中的不同的值）
那么我们可以这么理解：
浅拷贝（shallowCopy）只是增加了一个指针指向已存在的内存地址，
深拷贝（deepCopy）是增加了一个指针并且申请了一个新的内存，使这个增加的指针指向这个新的内存，
使用深拷贝的情况下，释放内存的时候不会因为出现浅拷贝时释放同一个内存的错误。
