webpack4.0安装及常用配置

发表于 2020-04-24 | 分类于 前端 |

Webpack 是一个前端资源加载/打包工具。它将根据模块的依赖关系进行静分析，然后将这些模块按指定的规则生成对应的静态资源。本文将对常用的一些配置进行详细的解释，如果项目中需要其他的配置，请登陆webpack官网进行查询

第一章 webpack安装

  • 中文官网 https://www.webpackjs.com

  • 安装本地webpack
    命令：npm install --save-dev webpack webpack-cli

    webpack 默认0配置，打包src内部文件，输出到dist目录

  • 创建文件夹，在文件夹目录下运行npm install –sava-dev webpack webpack-cli 安装本地webpack

    此提示是指可设置webpack的mode属性，可以是development(开发者模式)或production(生产者模式).
    例如：npx webpack –mode development

  • 打包后生成dist文件夹，文件夹内生成main.js文件，我们在dist文件夹中添加index.html文件，引入main.js

  • 浏览器中打开index.html，打开控制台，可以看到我们src文件中的index.js中代码的输出

    第五章 图片处理

    在项目中，我们会使用很多静态资源的图片，或者大量的小图标图片，在指定图片size，或者小图标图片过多的时候，就需要我们使用url-loader及file-loader(由于版本不同，在url-loader执行不成功时，需要安装file-loader)，将其转换成base64格式存放在一个文件中，这样会比大量的静态资源方便很多

  • 安装命令：

  • 运行npm run build打包后，可以看到我们的图片已经变为base64格式了
