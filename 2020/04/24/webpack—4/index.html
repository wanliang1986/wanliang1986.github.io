webpack—4讲解

发表于 2020-04-24 | 分类于 前端

Webpack 是一个前端资源加载/打包工具。它将根据模块的依赖关系进行静分析，然后将这些模块按指定的规则生成对应的静态资源。

第一章 webpack安装

  • 中文官网 https://www.webpackjs.com

  • 安装本地webpack
    命令：npm install --save-dev webpack webpack-cli

    webpack 默认0配置，打包src内部文件，输出到dist目录

  • 创建文件夹，在文件夹目录下运行npm install –sava-dev webpack webpack-cli 安装本地webpack

  • 创建src文件夹，文件夹下添加index.js文件，文件内容如下

  • 运行命令: npx webpack

    此提示是指可设置webpack的mode属性，可以是development(开发者模式)或production(生产者模式).
    例如：npx webpack –mode development

  • 打包后生成dist文件夹，文件夹内生成main.js文件，我们在dist文件夹中添加index.html文件，引入main.js

  • 浏览器中打开index.html，打开控制台，可以看到我们src文件中的index.js中代码的输出

    html插件

    html插件的作用是在打包后运行打包后文件中的html文件时，url地址并不是通过文件路径的方式打开，而是根据配置进行localhost：端口号的方式或者ip地址的方式打开

    • 运行命令：npm install --sava-dev webpack-dev-server
    • 在项目的package.json文件中scripts对象中添加配置
    • 在webpack.config.js文件中配置devServer

      此时即可使用npm run dev来运行，打开命令行中对应的http链接，即可看到用配置的ip及端口号的页面

    虽然我们完成了ip及端口号的配置，但是此时打包文件中的html文件是我们手动生成的，如果没有这个html文件，那么生成的对应端口的页面会报错，下面将对自动生成html页面进行配置

    • 在src文件夹中新建一个html文件，作为打包生成html文件的模板
    • 安装HtmlWebpackPlugin插件
      npm install --save-dev html-webpack-plugin
    • 在webpack.confing.js中引入并进行插件的配置
    • 删除原有的打包文件，重新运行打包命令，npm run build,可以发现打包文件中自动生成了index.html文件
