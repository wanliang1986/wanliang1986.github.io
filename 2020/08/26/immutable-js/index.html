immutable.js

发表于 2020-08-26 | 分类于 前端 |

介绍

immutable与react同期出现，但没有被默认放到react工具集中。它内部实现了一套完整的数据持久化体系，同时还有很多比较容易使用的数据类型，
比如 List,Map,Set等
其中，最重要的三种数据结构：

  • Map: 键值对集合，对应于Object

  • List: 有序可重复的列表，对应于Array

  • Set：无序不可重复的列表
    immutable Data 一旦创建，就不能再被更改的数据，对immutable对象的任何修改和添加删除操作，都会返回一个新的immutable对象，这是其最本质的特点

    为什么数据不能被改变，这是由于我们所运用的场景，比如redux来决定的， redux的基本原则是redux里的数据都是只读的，所以如果在redux中要对数据进行操作，那么就需要对原数据进行拷贝操作，这时候就可以用到immutable；另一种情况是在一个复杂的业务场景中，存在一个共享数据，当多个业务场景都可以操作这个数据时，就会出现不可控的结果。

    优点：

  • 降低Mutable带来的复杂度（只针对要修改的节点进行了拷贝）

  • 降低了内存占用

  • 便于函数式编程

    缺点：容易与原生对象搞混

常见API

API_List

首先，List对应原生js的数组，但是List是工厂方法，所以不能用new初始化

常用方法：

List.isList（）与List.of（）

  • List.isList()判断是否是List类型
  • List.of()创建新的List

size获取长度与set方法

  • size 取得List的长度
  • set方法用来设定指定下标的值 set（下标，值）,下标可以超过List数组的最大长度，中间空白的位置会自动进行填充

delete，insert,update,clear,setSize,setIn

  • delete删除指定下标的值
  • insert 插入值
  • update 更新指定下标的值
  • clear 清空并返回一个长度为0的数组
  • setSize 重新设定数组长度，小于原数组长度会截断，大于原数组长度会用undefined进行填充
  • setIn()用来设定嵌套结构的值([第一层下标，第二层下标，…第N层下标]，值),二维数组
  • List可用原生数组的所有方法，一些特殊的方法可查官网Api

API_MAP

Map对应原生js的Object结构，它是无序的
Map是工厂发给发，不能使用new 实例化

set,get,delete,deleteAll，clear

  • set设定值
  • get取值
  • delete删除值
  • deleteAll批量删除
  • clear清楚所有，返回新的Map

update,merge,concat,mergeWith,setIn

  • update 更新 update(key,callback)
  • merge与concat相同，合并
  • mergeWith 类似与merge,但指定了具体规则
  • setIn 对于嵌套结构进行设置

toJs与toJSON，将Map转换成原生object，深转换与浅转换

  • toJs转换全部结构
  • toJSON只转换一层

API_SET

  • union 将多个Set合并成一个Set
  • intersect 取N个Set的交集
  • subtract 从set中除去一些值
  • groupBy 分组
